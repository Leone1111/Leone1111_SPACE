## 2. 文档结构

| 功能 | Markdown 格式 |
|------|---------------|
| 一级标题 | `# 标题` |
| 二级标题 | `## 标题` |
| 三级标题 | `### 标题` |
| 四级标题 | `#### 标题` |
| 五级标题 | `##### 标题` |
| 六级标题 | `###### 标题` |
| 一级标题另一种写法 | `标题下一行使用 ===` |
| 二级标题另一种写法 | `标题下一行使用 ---` |
| 自动目录 | `[TOC]` |
| 水平分割线 | `---` |
| 水平分割线 | `***` |
| 水平分割线 | `___` |

### 标题使用规则

- `#` 后面建议加一个空格。
- 标题层级建议按顺序使用，不要从一级标题直接跳到三级标题。
- 一篇文档通常只保留一个一级标题。
- 标题尽量短，便于生成目录和页面锚点。

## 3. 段落与换行

| 功能 | Markdown 格式 |
|------|---------------|
| 新段落 | 两段之间空一行 |
| 强制换行 | 行尾添加两个空格后换行 |
| 强制换行 | `<br>` |
| 部分编辑器支持的换行 | 行尾使用 `\` |

### 段落使用规则

- 普通段落不需要特殊符号。
- 不建议用空格或 Tab 缩进普通段落。
- 标题、列表、代码块、表格前后建议保留空行。
- 多段文字之间用空行分隔。

## 4. 文本样式

| 功能 | Markdown 格式 |
|------|---------------|
| 粗体 | `**文本**` |
| 粗体 | `__文本__` |
| 斜体 | `*文本*` |
| 斜体 | `_文本_` |
| 粗斜体 | `***文本***` |
| 粗斜体 | `___文本___` |
| 删除线 | `~~文本~~` |
| 下划线 | `<u>文本</u>` |
| 高亮 | `==文本==` |
| 高亮 HTML 写法 | `<mark>文本</mark>` |
| 行内代码 | `` `代码` `` |
| 键盘按键 | `<kbd>Ctrl</kbd>` |
| 上标 | `<sup>文本</sup>` |
| 下标 | `<sub>文本</sub>` |

### 文本样式使用规则

- 粗体和斜体建议优先使用 `*`，兼容性较好。
- 删除线、高亮、下划线属于扩展语法，不是所有平台都完全支持。
- 需要更精确的格式控制时，可以插入 HTML 标签。

---

## 5. 列表

| 功能 | Markdown 格式 |
|------|---------------|
| 无序列表 | `- 内容` |
| 无序列表 | `* 内容` |
| 无序列表 | `+ 内容` |
| 有序列表 | `1. 内容` |
| 子级列表 | 在子项前缩进 2 到 4 个空格 |
| 未完成任务 | `- [ ] 内容` |
| 已完成任务 | `- [x] 内容` |

### 列表使用规则

- 推荐统一使用 `-` 作为无序列表标记。
- 列表符号后面必须有一个空格。
- 子列表缩进应保持一致。
- 嵌套层级不宜过深，通常控制在 3 层以内。
- 有序列表的数字可由 Markdown 自动调整，但建议手动保持连续。

## 6. 引用块

| 功能 | Markdown 格式 |
|------|---------------|
| 单行引用 | `> 引用内容` |
| 多行引用 | 每行前加 `>` |
| 引用内空行 | `>` |
| 二级引用 | `>> 引用内容` |
| 三级引用 | `>>> 引用内容` |
| 引用中使用列表 | `> - 内容` |
| 引用中使用标题 | `> ## 标题` |
| 引用中使用代码 | `> \`代码\`` |

### 引用使用规则

- `>` 后面建议加一个空格。
- 引用块可以嵌套。
- 引用块中可以放标题、列表、代码、链接和图片。
- 在列表中嵌套引用时，需要额外缩进。

---

## 7. 链接

| 功能 | Markdown 格式 |
|------|---------------|
| 普通链接 | `[链接文字](链接地址)` |
| 带标题的链接 | `[链接文字](链接地址 "标题")` |
| 自动链接 | `<链接地址>` |
| 邮箱链接 | `[发送邮件](mailto:邮箱地址)` |
| 电话链接 | `[拨打电话](tel:电话号码)` |
| 参考式链接 | `[链接文字][标签]` |
| 参考链接定义 | `[标签]: 链接地址 "标题"` |
| 简化参考链接 | `[链接文字][]` |
| 简化参考链接定义 | `[链接文字]: 链接地址` |

### 链接使用规则

- 普通链接适合短链接和单次使用。
- 参考式链接适合长文档、多次引用同一链接的情况。
- 自动识别 URL 的能力取决于具体 Markdown 编辑器。
- 为保证兼容性，建议优先使用标准链接语法。

---

## 8. 图片

| 功能 | Markdown 格式 |
|------|---------------|
| 普通图片 | `![替代文字](图片路径)` |
| 带标题图片 | `![替代文字](图片路径 "图片标题")` |
| 本地相对路径图片 | `![替代文字](./images/图片名.png)` |
| 网络图片 | `![替代文字](图片网址)` |
| 参考式图片 | `![替代文字][标签]` |
| 参考式图片定义 | `[标签]: 图片地址` |
| 指定图片宽度 | `<img src="图片地址" width="50%">` |

### 图片使用规则

- 推荐使用相对路径，便于网站或项目迁移。
- 建议单独建立 `images/` 或 `assets/` 文件夹存放图片。
- 图片替代文字应简洁说明图片内容。
- Markdown 原生语法一般不能控制图片宽高，需要用 HTML。
- 网络图片要注意链接稳定性、加载速度和版权问题。

## 9. 代码

| 功能 | Markdown 格式 |
|------|---------------|
| 行内代码 | `` `代码` `` |
| 行内代码包含反引号 | 使用双反引号包裹 |
| 缩进式代码块 | 每行前加 4 个空格或 1 个 Tab |
| 普通代码块 | 使用三个反引号包裹 |
| 指定语言代码块 | ` ```语言名 ` |
| 差异代码块 | ` ```diff ` |

### 常用代码语言标识

| 语言 | 标识 |
|------|------|
| Python | `python` 或 `py` |
| JavaScript | `javascript` 或 `js` |
| TypeScript | `typescript` 或 `ts` |
| HTML | `html` |
| CSS | `css` |
| Shell | `bash` 或 `shell` |
| SQL | `sql` |
| JSON | `json` |
| YAML | `yaml` 或 `yml` |
| Java | `java` |
| C++ | `cpp` 或 `c++` |
| C# | `csharp` 或 `c#` |
| PHP | `php` |
| Ruby | `ruby` 或 `rb` |
| Go | `go` |
| Rust | `rust` |

### 代码使用规则

- 多行代码推荐使用三个反引号代码块。
- 指定语言后，支持语法高亮的平台会自动高亮。
- 缩进式代码块前后建议留空行。
- 在列表中嵌套代码块时，缩进需要更深。
- 行号显示、代码高亮、差异对比属于平台扩展能力。

## 10. 表格

| 功能 | Markdown 格式 |
|------|---------------|
| 表格列分隔 | 使用 `\|` |
| 表头分隔 | 使用 `---` |
| 左对齐 | `:---` |
| 右对齐 | `---:` |
| 居中对齐 | `:---:` |
| 表格中显示竖线 | `\|` |
| 表格中换行 | `<br>` |
| 表格中使用代码 | `` `代码` `` |
| 表格中使用链接 | `[文字](链接)` |
| 表格中使用粗体 | `**文本**` |

### 表格使用规则

- 表头和内容之间必须有分隔线。
- 分隔线至少使用三个连字符。
- 表格两侧的竖线不是绝对必须，但建议保留。
- 表格不要求严格对齐，但源码对齐后更容易维护。
- 表格中的复杂内容建议适当简化，避免太长。

## 11. 数学公式

| 功能 | Markdown / LaTeX 格式 |
|------|-----------------------|
| 行内公式 | `$公式$` |
| 块级公式 | `$$公式$$` |
| 上标 | `x^2` |
| 下标 | `x_1` |
| 多字符上标 | `x^{n+1}` |
| 多字符下标 | `x_{i+1}` |
| 分数 | `\frac{分子}{分母}` |
| 平方根 | `\sqrt{x}` |
| n 次根 | `\sqrt[n]{x}` |
| 求和 | `\sum` |
| 连乘 | `\prod` |
| 积分 | `\int` |
| 希腊字母 | `\alpha`、`\beta`、`\gamma` |
| 多行对齐公式 | `align` 环境 |

### 常用数学符号

| 类型 | 格式 |
|------|------|
| 乘号 | `\times` |
| 除号 | `\div` |
| 不等于 | `\neq` |
| 恒等于 | `\equiv` |
| 小于等于 | `\leq` |
| 大于等于 | `\geq` |
| 约等于 | `\approx` |
| 相似 | `\sim` |
| 属于 | `\in` |
| 不属于 | `\notin` |
| 子集 | `\subset` |
| 超集 | `\supset` |
| 交集 | `\cap` |
| 并集 | `\cup` |
| 空集 | `\emptyset` |

### 数学公式使用规则

- 数学公式依赖编辑器是否支持 KaTeX 或 MathJax。
- 简短公式用行内公式。
- 重要推导、复杂公式用块级公式。
- 多字符上下标必须使用 `{}` 分组。

## 12. Mermaid 图表

| 功能 | Markdown 格式 |
|------|---------------|
| Mermaid 图表代码块 | ` ```mermaid ` |
| 流程图 | `graph TD` |
| 时序图 | `sequenceDiagram` |
| 甘特图 | `gantt` |
| 类图 | `classDiagram` |
| 状态图 | `stateDiagram` |
| 饼图 | `pie` |

### 流程图方向

| 方向 | 含义 |
|------|------|
| `TD` | 从上到下 |
| `TB` | 从上到下 |
| `BT` | 从下到上 |
| `LR` | 从左到右 |
| `RL` | 从右到左 |

### 流程图节点

| 节点格式 | 含义 |
|----------|------|
| `A[文本]` | 矩形节点 |
| `A(文本)` | 圆角矩形节点 |
| `A{文本}` | 判断节点 |
| `A((文本))` | 圆形节点 |
| `A>文本]` | 旗帜形节点 |

### 流程图连接线

| 连接线 | 含义 |
|--------|------|
| `-->` | 实线箭头 |
| `-.->` | 虚线箭头 |
| `==>` | 粗实线箭头 |
| `--` | 实线 |
| `-.` | 虚线 |

### 时序图常用格式

| 功能 | Mermaid 格式 |
|------|--------------|
| 定义参与者 | `participant A as 名称` |
| 实线箭头 | `A->>B: 内容` |
| 虚线箭头 | `A-->>B: 内容` |
| 注释 | `Note right of A: 内容` |
| 循环 | `loop 条件` |
| 结束块 | `end` |

### 甘特图常用格式

| 功能 | Mermaid 格式 |
|------|--------------|
| 甘特图开始 | `gantt` |
| 标题 | `title 标题` |
| 日期格式 | `dateFormat YYYY-MM-DD` |
| 分组 | `section 分组名` |
| 已完成任务 | `:done` |
| 进行中任务 | `:active` |
| 关键任务 | `:crit` |
| 依赖前一任务 | `after 任务ID` |

### Mermaid 使用规则

- Mermaid 是否可用取决于编辑器支持。
- GitHub、Obsidian、Typora、部分在线编辑器支持程度不同。
- 图表语法建议单独放在代码块中。
- 复杂图表建议先在支持 Mermaid 的编辑器中预览。

## 13. 脚注

| 功能 | Markdown 格式 |
|------|---------------|
| 脚注引用 | `[^标签]` |
| 脚注内容 | `[^标签]: 脚注说明` |

### 脚注使用规则

- 脚注常用于补充说明、文献说明、术语解释。
- 脚注标签可以是数字、英文或简短文本。
- 脚注内容通常放在文档底部。
- 脚注属于扩展语法，并非所有平台都支持。

## 14. HTML 扩展

| 功能 | HTML 格式 |
|------|-----------|
| 换行 | `<br>` |
| 下划线 | `<u>文本</u>` |
| 高亮 | `<mark>文本</mark>` |
| 键盘按键 | `<kbd>按键</kbd>` |
| 加粗 | `<b>文本</b>` |
| 斜体 | `<i>文本</i>` |
| 强调 | `<em>文本</em>` |
| 上标 | `<sup>文本</sup>` |
| 下标 | `<sub>文本</sub>` |
| 图片大小控制 | `<img src="图片地址" width="宽度">` |

### HTML 使用规则

- Markdown 中可以直接插入部分 HTML 标签。
- HTML 适合解决 Markdown 原生语法做不到的格式控制。
- 不同平台对 HTML 的支持程度不同。
- 在 GitHub、微信公众号、部分在线编辑器中，部分 HTML 可能被过滤。

## 15. 转义字符

| 需要显示的字符 | Markdown 转义格式 |
|----------------|-------------------|
| 反斜杠 | `\\` |
| 反引号 | `` \` `` |
| 星号 | `\*` |
| 下划线 | `\_` |
| 花括号 | `\{ \}` |
| 方括号 | `\[ \]` |
| 小括号 | `\( \)` |
| 井号 | `\#` |
| 加号 | `\+` |
| 减号 | `\-` |
| 英文句点 | `\.` |
| 感叹号 | `\!` |
| 竖线 | `\|` |

### 转义使用规则

- 当特殊符号被 Markdown 识别为格式符号时，需要转义。
- 表格中如果要显示竖线 `|`，通常需要写成 `\|`。
- 代码块中的大多数特殊符号不需要转义。
- 行内代码适合展示命令、路径、变量名和符号。

## 16. 常见写作规范

| 场景 | 建议 |
|------|------|
| 文档标题 | 一个文档尽量只用一个一级标题 |
| 小节标题 | 按层级顺序使用，不跳级 |
| 段落 | 段落之间空一行 |
| 列表 | 同一文档中统一列表符号 |
| 表格 | 表头、分隔线、内容行结构保持完整 |
| 代码 | 多行代码用代码块，不要用普通段落模拟 |
| 图片 | 使用有意义的替代文字 |
| 链接 | 重要链接建议使用标准链接格式 |
| 公式 | 复杂公式单独成块 |
| 图表 | Mermaid 图表需要确认编辑器支持 |
| HTML | 只在 Markdown 原生语法不够时使用 |
| 转义 | 显示特殊符号时使用反斜杠 |
